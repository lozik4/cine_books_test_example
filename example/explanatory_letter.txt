Серед всіх тестів є 1 який спеціально зроблений як помилковий для демонстрації звіту.
Відео занадто велике навіть для того щоб його завантажити в git lfs томуму даю лінку з гугл драйву.
відео із звуком в кінці є деякі пояснення

https://drive.google.com/file/d/1VC6DaPyGEJNMcRNgVJkHDFQoNFl2gsnN/view?usp=share_link

Також в цій діректорії є скріншоти звіту для наглядності.

Всі команди які я виконую в терміналі виконуються завдяки CI я звик використовувти Jenkins
але зараз дивлюся в напрямку TeamCity ще вивчаю данний інструмент.

У тестів є змога запускатись паралельно для того щоб пришвидшити прогон це дуже відчутно коли тестів
вже стає багато.

Також якщо відключити інтерфейс також швидкість виросте відсотків на 15-20% така функція закладена
--headles True в командну строку додати. Також не завжди я підіймаю новий бразер для кожного тесту.

Для менеджменту браузерів локально я використовую Вебдрайвер менеджер, для CI використовую SeleniumGrid
файл для докеру docker-compose-grid.yml і при запуску прапорець --hub True

Більше інформації про структуру і можливості фреймворку можна прочитати в README.md
